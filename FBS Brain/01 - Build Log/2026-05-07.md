### PWA Auto-Update Pipeline (3-layer fix)
Root cause: users on faithbeyondsundays.app were seeing stale app versions on iOS Safari because three caching layers were broken simultaneously.

**Layer 1 ‚Äî Vercel CDN (`vercel.json`)**
- `source: "/index.html"` does NOT match requests to `/` ‚Äî had to add explicit `source: "/"` rule
- Both `/` and `/index.html` now get `no-cache, no-store, must-revalidate`
- `/sw.js` also no-cache
- `/assets/*` set to immutable (these are content-hashed, safe)

**Layer 2 ‚Äî Service Worker HTML precaching (`vite.config.ts`)**
- `globPatterns` included `html` ‚Äî SW was caching index.html and serving it stale, bypassing Vercel headers entirely
- Removed `html` from glob: `["**/*.{js,css,ico,png,svg,woff2}"]`
- HTML is now never in the SW cache ‚Äî always fetched from network

**Layer 3 ‚Äî Page reload trigger (`main.tsx`)**
- `registerSW({ onNeedRefresh })` never fires with `registerType: "autoUpdate"` ‚Äî wrong API
- Replaced with `controllerchange` event listener + `hadController` guard to prevent reload loop on first SW install
- Now reloads exactly once when a new SW takes control

### PWA Icon ‚Äî Church Logo on Home Screen
- `index.html` had `<link rel="apple-touch-icon" href="/icons/icon-192x192.png">` ‚Äî iOS gives this absolute priority over anything in the manifest, completely overriding the dynamic per-church icon
- Removed that tag; iOS now falls through to manifest icons
- `apple-mobile-web-app-title` changed from `"Faith Beyond Sundays"` to `""` so manifest `short_name` wins
- Tested: Overflow Church PWA icon now shows OVF logo ‚úÖ

### Church Name Fix
- `api/manifest.ts` had `.slice(0, 12)` on `short_name` ‚Äî caused "Overflow Church" ‚Üí "Overflow Chu"
- Removed the truncation; short_name is now first 2 words with no length cap
- Tested: shows "Overflow Church" in full ‚úÖ
