### 2. Group Chat UX Overhaul

Full iMessage-style chat experience. All changes in `GroupChat.tsx`, `GroupPage.tsx`, `GroupsListScreen.tsx`, `Index.tsx`.

**Layout:**
- Chat takes over full screen (`position: fixed; height: 100dvh; z-index: 60`)
- Nav bar hidden while in chat (`overlay === "community-groups"` check in Index.tsx)
- Sticky group name header with correct top padding (matching all other page headers)
- Input bar always visible — no scrolling required to reach it
- Input bar bottom padding matches nav bar safe area (`env(safe-area-inset-bottom, 0px)`)

**Input:**
- Changed from `<input>` to auto-resizing `<textarea>` (grows vertically, max 120px — like iMessage)
- Enter sends, Shift+Enter adds a new line

**Messages:**
- iMessage-style day separators: "Today" / "Yesterday" / "Monday" / "Thu, Apr 23"
- No per-message timestamps — swipe left on any message row to reveal time
- Fixed scroll/swipe conflict: swiping left while scrolled up was jumping to bottom. Fixed with native touch event listeners (`{ passive: false }` on `touchmove`) + direction lock (first 4px of movement determines horizontal vs vertical — locks in that direction, prevents the other)

**Demo data:**
- Women's Group timestamps updated to span today / yesterday / 2 days ago (Mon) / 7 days ago (Thu Apr 23)

### 3. Tap-to-Profile

Tapping any other member's avatar or name opens their `PublicProfileScreen`.

**In group chat:** Avatar circle and sender name above bubble are now tappable buttons. Own messages are non-interactive (disabled).

**In group members list:** Each row in the Members section of the group landing page is now a tappable button.

**Profile fetch:** Two separate queries (profiles_safe → then churches by church_id) instead of a join, which was returning 400 on the view.

**Flow:** `onViewProfile(userId)` → fetch profile → construct `CommunityMember` → `setSelectedMember` + `setSubOverlay("public-profile")` + `window.history.pushState`. Back arrow triggers `popstate` → clears subOverlay → returns to groups.
