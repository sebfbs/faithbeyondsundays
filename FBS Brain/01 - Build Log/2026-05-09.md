# Build Log — 2026-05-09 (Session 13)

## What We Built

### Verified Profile Badge — Full Flow ✅

**Problem:** Every new user was auto-verified because Supabase sets `email_confirmed_at` automatically when "Confirm email" is OFF. The badge was unlocked for everyone from the moment they signed up.

**Solution:** Replaced `email_confirmed_at` with a `manually_verified` column that starts `false` for everyone and only becomes `true` when a user completes the email verification flow.

---

### Changes Made

**SQL (run in Supabase):**
```sql
ALTER TABLE profiles ADD COLUMN manually_verified boolean NOT NULL DEFAULT false;
```

**`src/components/fbs/ProfileScreen.tsx`:**
- Replaced `const isEmailVerified = !!(authUser?.email_confirmed_at)` with a React Query fetch of `profiles.manually_verified`
- `refetchOnWindowFocus: true` so badge updates when user returns to PWA after verifying
- Changed `emailRedirectTo` from `window.location.origin` to `${window.location.origin}/verify-profile`
- Fixed badge modal CTA button color from old orange `hsl(38, 100%, 47%)` to app amber `hsl(43, 78%, 61%)`

**`src/pages/VerifyProfilePage.tsx` (new file):**
- Standalone landing page at `/verify-profile`
- Waits for Supabase to exchange the magic link token from URL hash → sets `manually_verified = true`
- Shows three states: loading (spinner), success ("Profile Verified. You can close this tab."), error ("Link Expired")
- No app nav — minimal receipt-style page

**`src/App.tsx`:**
- Added `/verify-profile` route pointing to `VerifyProfilePage`

**`supabase/functions/_shared/email-templates/magic-link.tsx`:**
- Added `isVerification` boolean prop
- When true: heading = "Verify your profile", body = "Tap the button below to verify your profile.", CTA = "Verify my profile"
- Updated layout and button style to match signup email (white button, dark border, accent bar, proper container)

**`supabase/functions/auth-email-hook/index.ts`:**
- Detects verification magic links: `isVerification = emailType === 'magiclink' && redirectTo.includes('/verify-profile')`
- Overrides subject to "Verify your profile" when true
- Passes `isVerification` prop to template

---

## Full Flow (Tested + Working)

1. New signup → `manually_verified = false` → badge locked (gray, lock icon)
2. Tap badge → modal → tap "Verify Profile Now" → email sent
3. Email: subject "Verify your profile", CTA "Verify my profile"
4. Tap CTA → `/verify-profile` opens in browser
5. Page shows "Profile Verified. You can close this tab."
6. Return to PWA → badge auto-unlocks (React Query refetchOnWindowFocus)
