# Session 15 — 2026-05-11

## What We Built

### Churchless Signup — Eliminated by Design ✅
The root problem: users signing up without a church had no church linked to their profile. The solution: make it architecturally impossible.

**New routing:**
- `/?` (no church code) → Find Your Church screen
- `/?church=overflow` → goes straight into the app, church auto-linked on sign-up

**`joined_via` column added to `profiles`** — tracks whether a member came via a church link (`church_link`) or found the church through search (`direct_search`)

**Church ID persistence flow:**
- `/?church=overflow` lands → `Index.tsx` saves `fbs_church_id`, `fbs_church_name`, `fbs_church_code` to localStorage
- `ChurchLandingPage.tsx` (when used) also saves logo URL
- `OnboardingScreen.tsx` reads `fbs_church_id` from localStorage at profile creation time — church is linked atomically in the same insert

### Find Your Church Screen ✅
`src/components/fbs/FindChurchScreen.tsx` — shown when someone navigates to the root URL with no church code.
- Live search of all active churches (pulls from Supabase)
- Church logos + names
- "Don't see your church?" collapsible form → submits church request

### Church Request Flow ✅
- `church_requests` table in Supabase (church_name, requester_email, requester_name, contacted, created_at)
- RLS: anyone can insert, only platform admins can read/update
- `send-church-request` Edge Function → writes to DB + sends email to `sebastian@faithbeyondsundays.com` via Resend
- Platform dashboard → new "Church Requests" card at bottom with "Mark Contacted" toggle

### Auth Screen Church Branding ✅
When a member arrives via `/?church=overflow`, the auth screen shows:
- Church logo (or first-letter fallback)
- "Welcome to Overflow Church"
- "Stay connected to Sunday's message all week long"

### Church Onboarding Checklist ✅
`FBS Brain/04 - Operations/Church Onboarding Checklist.md` — 4-phase checklist (Platform Setup → Church Admin Setup → Internal Test → Go Live). Must be completed before sharing any church link with their congregation.

### ChurchLandingPage.tsx — Placeholder Built, Not Live ✅
Built `src/pages/ChurchLandingPage.tsx` with device detection (iOS Safari / Android / Chrome on iOS / Desktop). Currently not used — `/?church=overflow` routes to the app directly. The landing page design is a dedicated future session.

## SQL Run This Session
```sql
ALTER TABLE profiles ADD COLUMN IF NOT EXISTS joined_via TEXT DEFAULT 'church_link';

CREATE TABLE IF NOT EXISTS church_requests (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  church_name TEXT NOT NULL,
  requester_email TEXT NOT NULL,
  requester_name TEXT,
  contacted BOOLEAN DEFAULT FALSE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
ALTER TABLE church_requests ENABLE ROW LEVEL SECURITY;
-- + 3 RLS policies (insert anon, select platform_admin, update platform_admin)
```
