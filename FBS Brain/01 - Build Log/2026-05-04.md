- In production this won't matter for admins since they're invited via email

## Key Discoveries / Lessons

**Supabase PKCE OAuth:** Newer Supabase uses PKCE flow which puts `?code=` in the URL (not `#access_token` in the hash). Admin login was checking the hash only — this is why Google OAuth always fell through to the member app. Moot now that Google login is removed, but important to know.

**Email hook not configured in new project:** When trying to send a password reset email, got "Hook requires authorization token." The Resend + auth email hook setup from Session 1 was done on the old Lovable project. Needs to be re-configured on the new project before real churches can receive invite/reset emails.

**profiles.id vs profiles.user_id:** The `profiles` table has two UUID columns — `id` (auto-generated PK) and `user_id` (references `auth.users`). Any FK that needs to reference the auth user should point to `auth.users(id)` directly, not `profiles(id)`, unless you're doing a profile-to-profile join.
