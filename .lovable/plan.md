

## Generating Status Detail + Admin Approval Workflow

### Overview
Three changes: (1) animate the status badge icon when processing, (2) show what content is being generated by checking which `sermon_content` rows exist so far, and (3) add an approval step so admins can review all generated content before it goes live.

---

### 1. Animated Status Badges
When a sermon is in `transcribing` or `generating` state, the icon in the badge will spin using `animate-spin` with the `Loader2` icon, making it obvious the system is actively working.

### 2. Show What's Being Generated
For sermons in `generating` status, query the `sermon_content` table to see which content types have been created so far. Display a small progress list below the sermon card showing:
- Sparks -- done or pending
- Key Takeaways -- done or pending  
- Reflection Questions -- done or pending
- Scripture References -- done or pending
- Sermon Chapters -- done or pending

This uses the existing auto-polling (every 10s) to refresh, so items will check off as the backend generates them.

### 3. Admin Approval Workflow

**Database change:** Add a new status value `review` to the `sermon_status` enum, between `generating` and `complete`. Update the `process-sermon` edge function to set status to `review` instead of `complete` when all content is generated.

**Admin UI:** When a sermon is in `review` status:
- Show an "Approve" button on the sermon card
- Clicking "Approve" opens a review sheet/dialog showing all generated content (spark, takeaways, reflections, scriptures, chapters) so the admin can read through everything
- At the bottom, an "Approve & Schedule" button sets the sermon status to `complete` and marks it as published + current
- The admin can also go back if they want to wait

**Edge function change:** In `process-sermon/index.ts`, change the final status update from `complete` to `review` so the sermon waits for admin approval.

---

### Technical Details

**Migration SQL:**
```sql
ALTER TYPE public.sermon_status ADD VALUE 'review' BEFORE 'complete';
```

**Files modified:**
1. `src/pages/admin/AdminSermons.tsx` -- Add animated icons, content progress display for generating sermons, review status config, approve button, and review dialog with content preview
2. `supabase/functions/process-sermon/index.ts` -- Change final sermon status from `complete` to `review`

**Status flow after changes:**
```text
pending -> uploading -> transcribing -> generating -> review -> complete
```

The admin hits "Approve" on the review screen to move from `review` to `complete`, which then enables the publish/current controls that already exist.
